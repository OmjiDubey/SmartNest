❌ /live/relays is not retained.

State Cache
--------------------------------------------------------
Known limitation:
Current state is memory-backed.
Backend restart causes temporary state loss until MQTT updates arrive.
Recovery strategy will be added in a later phase.
---------------------------------------------------------


One Thing to Clarify with Hardware Team

Ask:

Are live/status and live/relays published with MQTT retain=true?

If not, ask whether they can add it.

This is usually a small firmware change and can eliminate most backend restart problems.

For SmartNest, retained messages on:

live/status
live/relays

would provide the highest value with the least complexity.
